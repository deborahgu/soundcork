Status guide
  • Live Marge is read from the speaker's current /info response. Soundcork means the speaker already points at this server, Bose means it still points at Bose cloud URLs, and Other means it points somewhere else.
  • Playback is inferred from the live /sources response. Soundcork-ready means this server is selected and radio sources are ready, Legacy-ready means the speaker still points at Bose but radio sources are still ready until a reboot, and Needs repair means internet radio sources are not currently ready.
  • REST /info means the speaker answered its HTTP API on port 8090. This is enough to inspect live state, but not enough to rewrite Bose URLs.
  • SSH means root shell access is available on port 22. Soundcork can copy the override XML and reboot the speaker.
  • Telnet CLIServer means the non-rooted SoundTouch command port 17000 is reachable. Soundcork can ask the speaker to update its Bose URLs and reboot.
  • Switch to Soundcork sets the Bose service URLs to this Soundcork instance. Use it when Live Marge is not Soundcork and either SSH or Telnet CLIServer is available.
